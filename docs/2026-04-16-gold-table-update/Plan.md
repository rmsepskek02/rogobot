# gold-table-update

- **날짜**: 2026-04-16 gold-table-update
- **담당**: developer / qa

## 목표
> !ㅂㄹ 명령어의 주급 골드 계산 테이블을 최신 레이드 기준으로 업데이트하고,
> 귀속골드(지평의 성당)를 주급과 별도로 표기한다.

## 작업 범위
- 수정 파일: `server/src/lostark-build.js`
- 영향받는 기능: `!ㅂㄹ`, `!배럭`, `!부캐`, `!ㅂㅋ`, `!원정대`, `!ㅇㅈㄷ` 명령어 출력

## 작업 계획

### 1. GOLD_TABLE 구조 변경
기존 `[최소레벨, 골드]` 2-tuple → `[최소레벨, 비귀속골드, 귀속골드]` 3-tuple 으로 변경

새 테이블 (골드 고정 기준):

| 최소레벨 | 비귀속 | 귀속 (지평) | 합계 |
|---------|--------|-----------|------|
| 1750    | 106,000 | 50,000   | 156,000 |
| 1740    | 106,000 | 40,000   | 146,000 |
| 1730    | 96,000  | 40,000   | 136,000 |
| 1720    | 82,000  | 40,000   | 122,000 |
| 1710    | 75,000  | 30,000   | 105,000 |
| 1700    | 60,000  | 30,000   | 90,000  |
| 1690    | 41,000  | 0        | 41,000  |
| 1680    | 35,500  | 0        | 35,500  |
| 1670    | 20,300  | 0        | 20,300  |
| 1660    | 13,600  | 0        | 13,600  |
| 1640    | 7,200   | 0        | 7,200   |

> 1690 이하는 지평의 성당이 없으므로 귀속골드 = 0. 기존 합계값 그대로 유지.

### 2. buildCharacters 함수 수정
- `gold` → `goldFree` (비귀속), `goldBound` (귀속) 두 변수로 분리
- `●주급 [${goldFree}G]` + `●귀속 [${goldBound}G]` 두 줄 출력
- 귀속골드가 0인 경우 귀속 줄 생략

## 완료 조건
- [ ] GOLD_TABLE이 3-tuple 구조로 변경됨
- [ ] 1750 이상 구간 신규 추가됨
- [ ] `●주급`은 비귀속 골드만 합산
- [ ] `●귀속`은 지평의 성당 골드만 합산하여 별도 출력
- [ ] 귀속골드가 0이면 귀속 줄 미출력

## 이슈 및 메모
> - 1690 이하 구간의 기존 합계값은 지평의 성당이 없으므로 전액 비귀속으로 처리
> - 기존 코드의 `mainCount === 6` 조건부 출력 로직은 그대로 유지
