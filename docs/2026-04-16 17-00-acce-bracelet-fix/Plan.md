# Acce Bracelet Fix

- **날짜**: 2026-04-16
- **담당**: developer / qa

## 목표

`!악세` 명령어에서 팔찌 효과가 출력되지 않는 버그를 수정하고, 각 효과를 `\n`으로 구분하여 출력한다.

## 현재 문제

1. **팔찌 효과 미출력**: `parseAccePiece` Loop 4에서 `t.value?.Element_000 === '팔찌 효과'`로 엄격 비교하지만, 실제 API 응답의 `Element_000` 값은 `"<FONT COLOR='#A9D0F5'>팔찌 효과</FONT>"` 형태라 조건이 항상 false
2. **효과 간 구분 없음**: 팔찌 효과가 출력되더라도 `\n` 없이 한 줄로 이어짐

## 기대 출력 형식

```
고대 팔찌 도약:18
 치명 119
 신속 83
공격 및 이동 속도가 4% 증가한다.
시드 등급 이하 몬스터에게 받는 피해량이 6% 감소한다.
```

## 작업 범위

- 수정 파일: `server/src/lostark-build.js`
- 수정 함수: `parseAccePiece` 내 Loop 4 (팔찌 효과 처리)

## 작업 계획

1. **조건 수정**: `=== '팔찌 효과'` → `.includes('팔찌 효과')`
2. **출력 정리**: `stripHtml`이 `<br>` → `\n` 변환을 이미 처리하므로, 각 줄을 `.split('\n')`으로 나눠 빈 줄 제거 후 각 줄 앞에 `\n` 추가
3. **기존 replace 체인 제거**: `stripHtml` + `\n` split으로 충분하므로 스탯 이름 replace 체인 불필요 (스탯 수치는 그대로 출력)

## 완료 조건

- [ ] `!악세` 응답에 팔찌 효과 출력됨
- [ ] 각 효과가 `\n`으로 구분됨
- [ ] `wrangler deploy --dry-run` 에러 없음

## 이슈 및 메모

- `추가 효과` 조건(`=== '추가 효과'`)도 동일한 HTML 태그 문제가 있을 수 있으나, 이번 범위는 팔찌 효과만 수정
- `stripHtml`이 `<br>` → `\n` 변환 후 나머지 태그를 제거하므로, API 응답의 개행은 이미 `\n`으로 처리됨
